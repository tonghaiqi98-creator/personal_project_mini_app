# 用户流程

## 顾客端流程

1. 顾客扫描桌号二维码。
2. 小程序打开并读取二维码参数。
3. 系统识别门店 ID 和桌号 ID。
4. 顾客进入菜单页。
5. 顾客按分类浏览菜品。
6. 顾客选择菜品并加入购物车。
7. 顾客进入购物车确认菜品、数量、金额和备注。
8. 顾客提交订单。
9. 系统创建订单，订单状态为待支付。
10. 顾客点击模拟支付。
11. 系统将订单状态更新为待接单。
12. 顾客进入订单状态页，等待商家接单和完成。

## 商家端流程

1. 商家打开商家端入口。
2. 商家查看今日订单列表。
3. 商家查看待接单订单。
4. 商家进入订单详情。
5. 商家确认订单内容。
6. 商家点击接单。
7. 系统将订单状态更新为制作中。
8. 菜品制作完成后，商家点击完成订单。
9. 系统将订单状态更新为已完成。
10. 今日统计数据同步更新。

## 订单状态流转

| 状态 | 状态值 | 触发动作 | 下一状态 |
| --- | --- | --- | --- |
| 待支付 | pending_payment | 顾客提交订单 | 支付成功后进入待接单 |
| 待接单 | paid | 模拟支付成功 | 商家接单后进入制作中 |
| 制作中 | accepted | 商家点击接单 | 商家完成后进入已完成 |
| 已完成 | completed | 商家点击完成订单 | 流程结束 |
| 已取消 | cancelled | 顾客或系统取消 | 流程结束 |

## MVP 流程说明

- 第一版支付为模拟支付，不调用微信支付。
- 第一版不处理退款。
- 第一版不处理配送状态。
- 第一版不处理多门店跨店订单。
- 第一版订单完成后只进入统计，不触发打印机。
